# Космо-Полёт

Гипер-казуальная игра в духе Flappy Bird, но в космической теме и с zero-G управлением. Зонд (Вояджер-1) летит сквозь поле астероидов в межзвёздном пространстве. Гравитации нет — каждый тап «пшикает» микро-двигателями и переключает направление вертикального движения. Бесконечный режим до первого проигрыша. Сделана с нуля без сторонних библиотек на чистом HTML/CSS/JS + Canvas.

## Что внутри

- **Управление одним тапом** (клик / Space / ↑) в стиле zero-G: 1-й тап — «пшик», зонд летит вверх; 2-й тап — «пшик», зонд летит вниз; и так далее. Никакой гравитации — постоянная скорость до следующего тапа.
- **Один режим — бесконечный**: играешь до первого проигрыша, после рестарта счёт обнуляется. Сложность плавно растёт.
- **Честная сложность** — генератор астероидов ограничивает разницу высоты прохода между парами, так что невозможных ситуаций не бывает.
- **Прогресс в localStorage**: рекорд, попытки, монеты, настройки звука и вибрации.
- **Реклама** (mock):
  - **Interstitial** показывается **перед началом следующей попытки** (не после смерти). Первые 2 попытки игрока без рекламы, затем — каждая 3-я (после 3 проигрышей).
  - **Rewarded** за вторую жизнь — только по тапу игрока на «Продолжить за рекламу».
  - Подключение к Яндекс.Рекламе подготовлено комментариями в `ads.js`.
- **Звук** — Web Audio API, синтез без файлов: тап, очко, столкновение, победа (на новом рекорде), покупка. Кнопка mute.
- **Вибрация** — короткая на очко, длиннее на крах. Можно отключить.
- **Графика** — всё рисуется кодом через Canvas. Параллакс из трёх слоёв звёзд, неоновая палитра, фиолетовая туманность, параболическая антенна и фольга на Вояджере, мини-облачко при «пшике» микро-двигателей.

## Запуск

ES-модули требуют HTTP-сервер (нельзя открывать `index.html` через `file://` — браузер заблокирует import-ы).

### Самое простое

```bash
# в папке с проектом:
python -m http.server 5173
# затем открыть http://localhost:5173 в браузере
```

### Альтернативы

- `npx serve .`
- VS Code расширение "Live Server"
- Любой статичный хостинг (GitHub Pages, Netlify, Vercel)

Для мобильного устройства — подключите ноутбук и телефон к одной Wi-Fi сети, на телефоне откройте `http://<ip-ноута>:5173`.

## Как поменять баланс

Все параметры — в [`config.js`](config.js). Например:

```js
thrustSpeed: 3.6,        // модуль вертикальной скорости после "пшика" (px/tick at 60fps)
thrustPuffMs: 320,       // длительность визуального облачка
obstacleSpeed: 2.6,      // стартовая скорость астероидов
obstacleGap: 175,        // стартовая высота прохода
difficultyGrowth: {
  speedPer10Points: 0.25,    // на сколько ускоряется каждые 10 очков
  gapShrinkPer10Points: 6,   // на сколько сужается проход
  maxSpeed: 5.5,
  minGap: 125,
}
```

Параметры рекламы тоже в `config.js`:

```js
ads: {
  mockAds: true,                 // переключи в false, чтобы использовать реальный SDK
  interstitialAfterDeaths: 3,    // показывать перед каждой попыткой после N смертей подряд
  firstAttemptsWithoutAds: 2,    // первые N попыток — без рекламы
  rewardedReviveEnabled: true,
}
```

## Как добавить скин

1. В [`skins.js`](skins.js) добавь запись в объект `SKINS`:

```js
export const SKINS = {
  default: { ... },
  ice: {
    id: 'ice',
    name: 'Ледяной',
    body: '#e0f2ff',
    bodyDark: '#7da3c4',
    nose: '#00d4ff',
    fin: '#80e8ff',
    window: '#0066aa',
    windowGlow: 'rgba(120, 230, 255, 0.9)',
    flameInner: '#cffcff',
    flameOuter: '#3aa0d0',
  },
};
```

2. Чтобы переключить скин — `Storage.set('skin', 'ice')`. В будущем добавится UI выбора в магазине (см. TODO.md).

## Как подключить рекламу (Yandex.Ads)

1. В [`config.js`](config.js) переключи `ads.mockAds: false`.
2. В [`ads.js`](ads.js) есть методы `showInterstitialAd()` и `showRewardedAd()` — там оставлены `// TODO: Yandex.Ads —` комментарии с примерным кодом вызова `ya.context.AdvManager.render(...)`. Замени mock-ветки на реальный SDK.
3. Подключи SDK Яндекс.Рекламы в `index.html` перед `<script type="module">`.
4. Правила показа (interstitial перед попыткой после N смертей, rewarded только по тапу игрока, защита первых попыток) реализованы в `AdManager.shouldShowInterstitialBeforeAttempt()` — менять не надо, они независимы от провайдера.

## Управление

| Клавиша / жест | Действие |
|---|---|
| Тап / клик / Space / ↑ | Прыжок |
| Esc / P | Пауза / продолжить |
| ⏸ кнопка | Пауза |

## Поддерживаемые браузеры

- Chrome / Edge / Safari (мобильные и desktop)
- Firefox
- Требуется поддержка ES Modules и Web Audio API. Для мобильных Safari аудио включается после первого пользовательского жеста (обработано).
