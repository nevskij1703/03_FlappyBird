# Космо-Полёт 🚀

Гипер-казуальная игра в духе Flappy Bird, но в космической теме. Ракета летит сквозь поле астероидов, игрок управляет высотой одним тапом — невесомость, гравитация и реакция. Сделана с нуля без сторонних библиотек на чистом HTML/CSS/JS + Canvas.

## Что внутри

- **Один тап / клик / Space** — импульс вверх. Между тапами ракету тянет гравитация.
- **Два режима**:
  - **Бесконечный** — плавно растущая сложность, рекорд хранится локально.
  - **Уровни** — 10 предустановленных уровней с фиксированными параметрами и целью.
- **Честная сложность** — генератор астероидов ограничивает разницу высоты прохода между парами, так что невозможных ситуаций не бывает.
- **Прогресс в localStorage**: рекорд, попытки, пройденные уровни, монеты, настройки звука и вибрации.
- **Реклама** (mock): interstitial после каждой 3-й смерти и каждого 2-го уровня (с защитой первые 2 попытки) + rewarded за вторую жизнь по тапу игрока. Подключение к Яндекс.Рекламе подготовлено комментариями в `ads.js`.
- **Звук** — Web Audio API, синтез без файлов: тап, очко, столкновение, победа, покупка. Кнопка mute.
- **Вибрация** — короткая на очко, длиннее на крах. Можно отключить.
- **Графика** — всё рисуется кодом через Canvas. Параллакс из трёх слоёв звёзд, неоновая палитра, фиолетовая туманность, мерцающее пламя ракеты.

## Запуск

ES-модули требуют HTTP-сервер (нельзя открывать `index.html` через `file://` — браузер заблокирует import-ы).

### Самое простое

```bash
# в папке с проектом:
python -m http.server 5173
# затем открыть http://localhost:5173 в браузере
```

### Альтернативы

- `npx serve .`
- VS Code расширение "Live Server"
- Любой статичный хостинг (GitHub Pages, Netlify, Vercel)

Для мобильного устройства — подключите ноутбук и телефон к одной Wi-Fi сети, на телефоне откройте `http://<ip-ноута>:5173`.

## Как поменять баланс

Все параметры — в [`config.js`](config.js). Например:

```js
gravity: 0.45,           // притяжение вниз (px/tick²)
jumpForce: -8.0,         // импульс прыжка (отрицательный = вверх)
obstacleSpeed: 2.6,      // стартовая скорость астероидов
obstacleGap: 175,        // стартовая высота прохода
difficultyGrowth: {
  speedPer10Points: 0.25,    // на сколько ускоряется каждые 10 очков
  gapShrinkPer10Points: 6,   // на сколько сужается проход
  maxSpeed: 5.5,
  minGap: 125,
}
```

Уровни — в [`levels.js`](levels.js). Каждый уровень — объект с `speed`, `gap`, `spacing`, `target` (сколько очков нужно для прохождения).

## Как добавить скин

1. В [`skins.js`](skins.js) добавь запись в объект `SKINS`:

```js
export const SKINS = {
  default: { ... },
  ice: {
    id: 'ice',
    name: 'Ледяной',
    body: '#e0f2ff',
    bodyDark: '#7da3c4',
    nose: '#00d4ff',
    fin: '#80e8ff',
    window: '#0066aa',
    windowGlow: 'rgba(120, 230, 255, 0.9)',
    flameInner: '#cffcff',
    flameOuter: '#3aa0d0',
  },
};
```

2. Чтобы переключить скин — `Storage.set('skin', 'ice')`. В будущем добавится UI выбора в магазине (см. TODO.md).

## Как подключить рекламу (Yandex.Ads)

1. В [`config.js`](config.js) переключи `ads.mockAds: false`.
2. В [`ads.js`](ads.js) есть методы `showInterstitialAd()` и `showRewardedAd()` — там оставлены `// TODO: Yandex.Ads —` комментарии с примерным кодом вызова `ya.context.AdvManager.render(...)`. Замени mock-ветки на реальный SDK.
3. Подключи SDK Яндекс.Рекламы в `index.html` перед `<script type="module">`.
4. Правила показа (interstitial каждую 3-ю смерть, rewarded только по тапу игрока, защита первых попыток) реализованы в `AdManager.shouldShowInterstitialOnDeath()` / `shouldShowInterstitialOnLevelEnd()` — менять не надо, они независимы от провайдера.

- Firefox
